# Заключение

- В ходе выполнения лабораторной работы были изучены механизмы SetUID, SetGID и Sticky-битов.
- **SetUID** позволяет процессу выполняться с правами владельца файла (`e_uid=0`).
- **Sticky-бит** на директории `/tmp` запрещает удаление файлов пользователям, не являющимся их владельцами.
- После снятия Sticky-бита удаление файла стало возможным для любого пользователя с правами на запись в директорию.
